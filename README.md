# AI Jeopardy!

A browser-based Jeopardy game where you compete against AI opponents. The board is generated fresh every game using GPT-4o-mini, and you buzz in with your voice.

![AI Jeopardy screenshot](https://via.placeholder.com/800x400?text=AI+Jeopardy!)

## Features

- **Voice buzz-in** — press BUZZ IN, then speak your answer (Chrome only, uses Web Speech API)
- **Fresh board every game** — 6 categories × 5 clues generated by GPT-4o-mini on every load
- **AI opponents** with distinct personalities, strategies, and accuracy levels
- **Clarification mechanic** — surname-only answers trigger a "Be more specific" follow-up when genuinely ambiguous
- **Realistic AI behavior** — AIs can buzz in, miss questions based on difficulty, and pick the next tile based on their strategy
- **Turn management** — whoever answers correctly controls the board, just like the real show

## How It Works

### Board Generation

On page load the game makes two rounds of API calls in parallel:

1. **GET `/api/categories`** — one fast call returns 6 category names
2. **6× POST `/api/category`** — all six run simultaneously, each returning 5 clues at $200–$1000

Categories appear with a loading state and fill in progressively as each parallel call resolves.

GPT is instructed to escalate difficulty from $200 (easy) to $1000 (hard/obscure), never include the answer in the clue text, and avoid the most obvious facts for each topic.

### Clue Reveal & Buzz Window

When you click a tile the clue reveals character-by-character at roughly 250 words per minute — simulating a host reading it aloud. Nobody can buzz in until the full clue has been revealed.

Once the reveal is complete:
- The BUZZ IN button unlocks
- A countdown bar drains over 8 seconds (human) / 5 seconds (AI re-buzz)
- AI opponents start their own randomized buzz timers based on their `speed` setting

### Answering

**Human:** Click BUZZ IN (or press Space), then speak your answer. A 5-second countdown ring gives you time to respond. Your spoken answer is transcribed live and sent to the judge when the timer expires.

**AI:** The AI "thinks" for a moment, then its answer is fetched from `/api/ai-answer`. Whether it answers correctly depends on its `accuracy` setting and the dollar value of the clue.

### Judging

Every answer goes through `/api/judge`, which uses GPT as a lenient Jeopardy judge:

- Accepts reasonable variations: adjectival forms (`Egyptian` → `Egypt`), nicknames, plurals, minor spelling differences
- Accepts a surname alone **unless** it's genuinely ambiguous for this specific clue — in which case it triggers the clarification mechanic
- Never reveals the correct answer when a player is wrong (other players may still buzz in)
- Returns three possible states: `correct`, `incorrect`, or `needsClarification`

### Clarification Mechanic

If you answer with just a surname that could plausibly refer to multiple well-known people (e.g., "Roosevelt" when the clue could be Teddy or Franklin), the judge sets `needsClarification: true`. You get 4 seconds to be more specific.

The clarification judge uses a stricter prompt:
- Accepts missing middle initials and common nicknames
- Rejects a wrong first name paired with the right surname
- Issues a final verdict — no further clarification rounds

This applies to both human and AI players.

### Turn Management

Whoever answers a question **correctly** controls the board and picks the next tile. If nobody gets a question right (all players wrong or the buzz timer expires), the player who picked the current tile retains control — matching the real show's rules.

The active controller is highlighted with a white glow on their score card and a "▶ your turn" label.

### Scoring

Correct answer: +`$value`  
Wrong answer: −`$value`  
No answer / timeout: `$0`

## AI Opponents

AI players are defined in [`public/players.json`](public/players.json):

```json
[
  { "name": "You", "isHuman": true },
  {
    "name": "HAL 9000",
    "isHuman": false,
    "avatar": "🤖",
    "strategy": "highValue",
    "speed": "fast",
    "accuracy": 0.90
  },
  {
    "name": "Deep Blue",
    "isHuman": false,
    "avatar": "♟️",
    "strategy": "sweepCategory",
    "speed": "medium",
    "accuracy": 0.75
  }
]
```

### Fields

| Field | Values | Description |
|---|---|---|
| `strategy` | `highValue`, `sweepCategory` | How the AI picks tiles. `highValue` always picks the highest remaining dollar value; `sweepCategory` finishes one category before moving on. |
| `speed` | `fast`, `medium`, `slow` | Controls buzz-in delay range. Fast AIs buzz in 0.8–2s after reveal; slow AIs 3–6s. |
| `accuracy` | `0.0` – `1.0` | Scales the chance of answering correctly. Combined with base miss rates per dollar value: `$200` 5% miss at accuracy 1.0, up to `$1000` 60% miss. |

Add more opponents by appending entries to `players.json` — no code changes needed.

## API Endpoints

| Method | Path | Description |
|---|---|---|
| GET | `/api/categories` | Returns 6 fresh category names |
| POST | `/api/category` | Returns 5 clues for a given category name |
| POST | `/api/ai-answer` | Generates an AI player's answer (supports clarification round) |
| POST | `/api/judge` | Judges a player's answer against the correct answer |

## Notes

- Voice recognition requires Chrome (Web Speech API is not supported in Firefox or Safari)
- Board generation costs ~7 GPT-4o-mini calls per game (1 for categories + 6 for clues)
- Answer judging costs 1–2 calls per answered clue (2 if clarification is triggered)
